## ⚠️ 先看:全局关键坑(这些最常卡人)

| 现象 | 真因 | 怎么办 |
|---|---|---|
| **游戏里听不到音频** | Wwise 事件对应的 **SoundBank 没生成**(`ak.soundengine` 只播放已生成 bank 里的内容) | 在 Wwise 里 **Generate SoundBanks**;auto-defined bank 会为新事件各建一个 .bnk |
| **P4 / Jira 连不上**(WSAEACCES / 超时) | 它们是**内网地址**;开着 **Mullvad VPN** 时全部流量走隧道,到不了内网。把 p4.exe 加进 split-tunnel 也没用 | **用 P4/Jira 时断开 Mullvad** |
| **Git push 卡住**(Connection reset :22) | GitHub SSH 22 端口被网络拦 | 已配置走 **443**(`~/.ssh/config` 指向 `ssh.github.com:443`),现在直接能推 |
| **Sound Finder 本地模型出垃圾结果** | 旧默认模型太小/没装 | 已统一用 **qwen2.5:7b-instruct**(本地 Ollama,GPU) |
| **动画音效(翅膀等)在游戏里没声** | ① clip 没被任何 Animator/controller 播放;② receiver 组件不在 Animator 所在 GameObject;③ 事件 bank 没生成 | 四件都要满足:clip 被播放 + .anim 有事件 + prefab 有 receiver(同 Animator 节点)+ bank 已生成 |

## 1. 音效检索与制作

### 1 · Sound Finder for Reaper
- **做什么**:本地音效库检索(172 万文件 FTS 索引),预览、推荐 layer 组合、拖进 Reaper。
- **怎么用**:点「模型拆解/搜索」用自然语言描述需求 → 生成分类+关键词 → 搜索 → 浏览/试听/拖入。
- **要点**:
  - 模型模式:`auto`(有 OPENAI_API_KEY 走远程,否则本地)/`local`(本地 Ollama qwen2.5:7b)。
  - **「每栏条数」**数字框:每个分类显示多少条(默认 80,可改;结果按"命中关键词数"排序,相关的在前)。
  - 描述要用**具体拟声词**(whoosh/swoosh/splash),少用抽象词(force/season),搜得更准。

## 2. Wwise 创建与测试

### 8 · Wwise Template Generator
- **做什么**:用审核过的模板,通过 WAAPI 在 Wwise 里批量创建对象结构(改名、Event 重定向、校验、dry-run)。
- **要点**:**目标父级路径**必须是当前打开 Wwise 工程里**真实存在**的对象;模板源对象也要在工程里。报错会弹「创建失败」对话框 + 报告(不再卡死)。

### 27 · Wwise Audio Logic Tester(LLM Sound Caster)
- **做什么**:用自然语言描述复杂播放逻辑(触发事件、定时调 RTPC、条件触发…)→ 本地/远程模型生成 DSL → WAAPI 驱动 `ak.soundengine` 实际出声测试。
- **怎么用**:连接 Wwise → 描述场景(可存入**场景库**复用)→ ① 生成 DSL → 检查/编辑 → ② 运行。
- **要点**:
  - 模型:`ollama`(本地)/ `openai` / `anthropic` / `claude-cli`·`codex-cli`(需装对应 CLI)。生成后日志会标**用的哪个模型**。
  - **出声前提**:目标事件的 bank 已生成;若是 Switch/Blend 容器(如呼吸),要先 `set_switch`(性别/视角等)+ `set_rtpc` 才有声。
  - setSwitch 参数名是 `switchState`(已修)。

### 28 · Audio Debug Assistant
- **做什么**:输入事件/Wwise 节点/问题描述,跑只读的 Wwise 本地检查 + Unity/Wwise 联检,给诊断。

### 21 · Animation Wwise Event AutoConfig
- **做什么**:给 Unity .fbx/.anim 配 Wwise 动画事件(如鸟翅膀下扑触发音效),并确保 prefab 有接收组件。
- **要点(重要)**:务必用 **Apply 模式**(报告 `Applied: True`),否则只算时间不写 receiver → 半配置静音。配完确认:clip 被 controller 播放、prefab 的 receiver 和 Animator 在**同一 GameObject**、事件 bank 已生成。

## 4. 提交流程 · 版本 · Jira

### 19 · P4V Changelist Organizer
- **做什么**:筛选/打标/学习/本地审阅 ProjectEF 音频相关待提交文件,分组进 P4 changelist(只用 opened/change/reopen/reconcile,不 submit/revert/delete)。
- **要点**:
  - 扫描已改为**后台线程**(不再卡死)。
  - 判定"音频相关"= 路径关键词(Wwise/Ak/Audio/Prefab)+ 扩展名(.prefab/.anim/.asset/.bnk/.wem…)+ 修改时间 + **AutoConfig 报告触碰索引**。
  - **P4 看不到磁盘改动**:直接写盘的文件要 `reconcile` 才进 CL。全量 reconcile 太慢 → 用**精准范围**(音频配置目标 + WwiseBanks + Bird 等目录)。
  - **游戏在另一台 P4**:`ef.p4.blackjack-local.com:1666`,client `yupeng_ADMIN-V9BNJMS5N`(不是 shell 默认的 Wwise 那台)。

### 26 · AudioTools Git Sync
- **做什么**:把音频工具源码 + Sound Finder 镜像进 GitHub 私有库,安全扫描(排除索引/DB/媒体/Cookie/密钥),提交,确认后推送。
- **怎么用**:① 扫描/同步 → ② 查看改动 → ③ 提交(本地)→ ④ 推送(二次确认)。push 已走 443,稳定。

### 22 · Audio Requirement Jira Triage(只读)
- **做什么**:扫设计文档成版本化的音频需求证据,匹配 Jira 单到具体设计位置。也是**登录 Jira 的地方**(cookie / Use Browser Login)。

### 34 · Audio Jira Creator(新)
- **做什么**:填**标题 + 工种** → 模板补全描述 + 工种映射 components → 预览 → **确认后**创建 PROEF 音频 Jira。
- **首次必做**:断开 Mullvad → 点「学习 Jira 规则」(拉合法 issue 类型/component)→ 复用 22 号的登录。
- **安全**:绝不自动建单,必须预览+确认;第一次建议用测试单验证字段。
